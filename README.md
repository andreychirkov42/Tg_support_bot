# Telegram support bot

Бот принимает сообщения клиентов и пересылает их в канал поддержки. Когда оператор отвечает в канале через Reply на сообщение от бота, ответ уходит обратно клиенту.

## Настройка

1. Создайте бота через [@BotFather](https://t.me/BotFather) и получите `BOT_TOKEN`.
2. Добавьте бота администратором в канал поддержки.
3. Скопируйте `.env.example` в `.env` и сначала заполните только `BOT_TOKEN`.
4. Запустите бота и опубликуйте в канале команду `/myid`, чтобы узнать `chat_id` канала.
5. Добавьте этот ID в `SUPPORT_CHAT_ID` и перезапустите бота.

```env
BOT_TOKEN=ваш_токен
SUPPORT_CHAT_ID=-1001234567890
DATABASE_PATH=support_bot.sqlite3
```

## Запуск

Нужен Python 3.10+.

```powershell
python bot.py
```

## Как пользоваться

- Клиент пишет боту любой текст, фото, файл, голосовое или другое обычное сообщение.
- В канале появляется карточка клиента и копия его сообщения.
- Оператор в канале нажимает Reply на карточку или на копию сообщения и пишет ответ.
- Бот отправляет ответ клиенту в тот же чат.

SQLite-файл `support_bot.sqlite3` нужен, чтобы бот помнил, какому клиенту соответствует каждое сообщение в канале. Telegram присылает боту только ID сообщения в канале, на которое ответили; исходный `chat_id` клиента нужно хранить отдельно. В этой же базе хранится offset обновлений, чтобы после перезапуска бот не обрабатывал старые сообщения повторно.
